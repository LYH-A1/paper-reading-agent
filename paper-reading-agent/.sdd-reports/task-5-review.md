## Additional Observations

- **Heuristic metadata extraction**: Title is taken as the first non-empty line of text, which is a reasonable heuristic for simple papers but may fail for multi-line titles or papers with front-matter.
- **Abstract regex fragility**: The pattern `r"Abstract\\s*\\n+(.+?)(?=\\n\\s*(?:\\d+\\.|[IVX]+\\.)\\s)"` assumes "Abstract" is followed by a newline and ends at the next section header. This may miss abstracts spanning multiple paragraphs without a following section header.
- **`_parse_pdfplumber` does not build sections**: Unlike `_parse_pymupdf`, the pdfplumber fallback only extracts raw text and metadata -- no section structure is produced.
- **Lazy imports**: Both `fitz` and `pdfplumber` are imported lazily inside the respective parse methods, which is good practice for startup performance.
- **Performance monitoring**: Parse duration >60s triggers a warning, which is sensible.
- **generate_sample.py fixture**: The `tests/fixtures/sample.pdf` was generated by a companion script `generate_sample.py` and correctly produces a document whose first page reads "Sample Paper Title" with an abstract and numbered sections, fulfilling the test expectations.

## Verdict
