## 第一阶段：验收测试（今天，1-2h）

**目标：** 用真实论文验证所有功能，记录问题。

### 准备

- [ ] 准备 3-5 篇不同领域的 arXiv PDF（计算机、生物、物理等）
- [ ] 准备 1 个 .bib 文件（从 Google Scholar 或 arXiv 导出）
- [ ] 打开 `http://localhost:8000`

### 测试场景

| # | 场景 | 预期结果 | 实际 |
|---|------|----------|------|
| 1 | 上传 PDF → 查看解析结果 | 标题/作者/章节正确 | |
| 2 | 问 "What is the main contribution?" | 回答含 R0 引用（Section X, Page Y） | |
| 3 | 问 "Summarize the method" | 结构化摘要，Section 引用 | |
| 4 | 问 "Compare this with related work" | 显示外部检索结果 [EXT-N] | |
| 5 | 保存 1 个外部结果到论文库 | 论文库出现新条目（External Save 标签） | |
| 6 | Import BibTeX | 导入成功，论文库显示 BibTeX Import 标签 | |
| 7 | 选 3 篇论文 → Compare | 生成结构化对比报告（方法/实验/贡献/局限） | |
| 8 | 论文库搜索 "transformer" | 只显示匹配论文，计数正确 | |
| 9 | 论文库筛选 External Save | 只显示外部保存的论文 | |
| 10 | 点一篇无 PDF 的论文 | 显示元数据卡片 + Upload PDF 按钮 | |

## 第三阶段：反馈收集（1-2 周）

**目标：** 让你自己或他人实际使用，发现设计层面的问题。

### 关注点

| 维度 | 问题示例 |
|------|----------|
| **准确性** | R0 引用是否真的指向正确段落？外部搜索结果是否相关？ |
| **速度** | 上传→可问答要多久？对比报告生成要多久？ |
| **易用性** | Compare 模式入口明显吗？保存按钮位置对吗？搜索够快吗？ |
| **完整性** | 缺什么功能让你想"要是能...就好了"？ |

### 记录方式

直接在 GitHub Issues 创建，打标签 `bug` / `enhancement` / `question`。

## 第四阶段：Phase 6 定义（2 周后）

**目标：** 基于真实反馈确定 Phase 6 范围。

### 候选方向（来自之前讨论）

| 方向 | 触发条件 |
|------|----------|
| **对比报告追问** | 如果用户多次在对比后问"那 A 和 B 在 XXX 上有什么区别？" |
| **arXiv PDF 一键下载** | 如果用户保存外部论文后频繁点 "Open on arXiv" |
| **Markdown 表格渲染** | 如果对比报告表格在聊天中显示为纯文本难读 |
| **响应式优化** | 如果在手机上打开体验差 |
| **论文库标签/分类** | 如果论文库超过 50 条后找不到想要的 |

### 决策方式

- 选 Issue 数最多 + 用户抱怨最频繁的方向
- 一次只做一个方向（保持 Phase 3-5 个 Task 的规模）
- 先出设计文档，确认后实现
